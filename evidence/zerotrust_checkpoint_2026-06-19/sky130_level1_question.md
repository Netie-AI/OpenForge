`OPENFORGE_SKY130_CARD=level1` uses the string constant `SKY130_MODELS_BUILTIN` in `openanalog/sim/models.py` (lines 22–31). That block is a hand-written level=1 SPICE `.model` card set with SKY130-style device names (`sky130_fd_pr__nfet_01v8`, `sky130_fd_pr__pfet_01v8`) and comment text "level-1 calibrated"; parameters (`vto=0.48`, `kp=270u` for NFET; `vto=-0.60`, `kp=90u` for PFET) differ from the bundled `nmos_ana`/`pmos_ana` cards in the same file (lines 15–19: `vto=0.7`, `kp=120u` NFET; `vto=-0.7`, `kp=40u` PFET). It is not loaded from `data/pdk/sky130/models.sp`, not fetched from the skywater-pdk repo at runtime, and not BSIM4 — `sky130_models_block()` returns `SKY130_MODELS_BUILTIN` when card flavor is `level1` (see `openanalog/sim/models.py` `sky130_models_block()` / `_sky130_card_flavor()`). The fetched BSIM path is only used when `OPENFORGE_SKY130_CARD=bsim`.
